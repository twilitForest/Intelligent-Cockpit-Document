一、二维码登录：

1.进入登录页，自动显示二维码，

2.用户扫描二维码后页面拉取app端该用户的头像和昵称，同时展示“请在手机上确认登录”提示和“返回重新登录”入口

3. 点击“返回重新登录”，即返回登录页，返回后二维码会自动刷新

4.用户扫描二维码后手机端未点击确认，1分钟后会自动返回登录页

5.二维码异常状态：

二维码展示60秒如未进行扫码登录即失效，静态展示“二维码已过期请点击刷新”

二维码获取失败，展示“二维码获取失败，请点击重新获取”

网络异常，展示“请您在网络良好时点击重新生码”

以上状态，点击二维码，即可刷新新的二维码。

二维码加载时，展示“加载中”样式，此时点击无响应。




二、手机短信登录：流程图如右侧

1.输入框内仅支持输入数字格式，非数字格式不可输入；

未输入手机号时点击登录或发送验证码，提示toast：手机号不能为空！

2.输入框自动校验手机号长度，超出11位为不可输入状态，点击“发送验证码”校验手机号长短，如不满足11位，toast提示“手机号格式不正确”；

3. 如手机号不正确，导致无法发送，toast提示“信息发送失败“

4. 如校验通过且手机号正常，判读该手机号1分钟内是否有再次发送验证码，如有，toast提示“1分钟内最多获取1次验证码”；

如没有，判读该发送验证码请求是否在5分钟内超过了3次，如超过3次，toast提示“5分钟内最多获取3次验证码”；

如5分钟内未超过3次，判断该发送验证码请求是否在1小时内超过了5次，如超过5次，toast提示“1小时内最多获取5次验证码”；

如1小时内未超过5次，判断该发送验证码请求是否在1天内超过了10次，如超过10次，toast提示“1天内最多获取10次验证码”；

5.如以上校验通过，则给该手机号发送验证码并toast提示“验证码发送成功”；同时“发送验证码”按钮进入60秒倒计时状态，显示“重新发送”，且该按钮为不可点击效果，超出60秒后，该按钮恢复为“发送验证码”效果，且可进行发送验证码操作

6.短信验证码发送已完成60秒倒计时，5分钟内填写该验证码进行登录，仍可校验成功，超出5分钟，该验证码失效，填写失效验证码提示toast:“验证码输入有误”

7.输入验证码，校验验证码是否正确，如验证码错误，toast提示“验证码输入有误”；如验证码正确，则toast提示“登录成功”
8.发送验证码后，按钮变为“重新发送（x秒）”样式，x秒-倒计时60s，结束后按钮重新变为“发送验证码”样式。

9.若登录成功，则提示toast“登录成功”并返回登录前页面，若登录前有订阅，购买操作，则继续响应订阅流程、购买流程。
